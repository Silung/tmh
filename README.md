# tmh / KuaiRec 事件与测试集

## 主流程在做什么

想从大量短视频 **caption** 里挖出「热门事件 / 话题」：每条视频对应多个标题式文本，希望聚合成**可解释的事件簇**，并用人话概括出来。做法是：先让模型从标题里抽**关键词**，再按关键词向量聚类，最后对每个簇做一次**事件归纳**（尽量落到具体人事时地，而不是空洞标签）。

技术路线一句话：**关键词 → 向量 → 聚类 → 每簇 LLM 归纳**。实现见 `main.py`，数据与接口配置见 `config.py`。

---

## 测试集在做什么

有了上面的挖掘方法之后，需要知道它**准不准**，就必须有可对照的标注。全库两两问模型「是不是同一事件」不可行（组合数爆炸），所以测试集要控制规模，同时尽量贴近真实任务：只关心**和内容相关、可能同属一个热点**的标题对，而不是随机乱对。

思路是：

1. **用同一套关键词结果当筛子**：只保留能挂上关键词的 caption；候选对限定为**至少共享一个提取出的关键词**（必要时再加少量「关键词完全不交」的对照组），这样边数可控、也更像「同题撞词」的真实检索/聚类场景。
2. **用 LLM 做主观但一致的成对判断**：每条候选对打一个 1–5 分，表达「是否同一热点/话题」；结果落库并支持断点续跑，避免长跑任务白做。
3. **从高分边拼出弱簇，再概括成「事件级 topic」**：在成对分数超过阈值的边上连成图，每个连通块视同一候选事件；对每个块再用 LLM 写成**短事件句**（谁做了什么 / 发生了什么），明显不成一则 **跳过**，避免用「娱乐」「商业」这类大类凑数。最终得到 **topic ↔ 多条 title** 的列表，便于人工扫一眼或接下游评测。

细节上（采样上限、并发、模型名、导出文件名等）以 `build_pairwise_evalset.py` 里说明和 `-h`/`list-models` 为准；常见顺序是：`sample` → `run` →（可选 `export`）→ `topics`（必要时再 `topics-list`）。
